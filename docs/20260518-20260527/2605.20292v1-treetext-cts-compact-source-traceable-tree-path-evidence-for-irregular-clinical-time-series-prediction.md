## 摘要
数值时间序列模型能够有效处理不规则的电子健康记录（EHR）轨迹，但无法自然地将支持每个风险估计的测量值和时序模式作为可读证据呈现。现有的基于文本的界面提高了可读性，但通常依赖原始序列化（冗长且冗余）或患者级别的自由形式摘要（难以追溯到源测量和时间窗口）。为弥合这一差距，我们引入了TreeText-CTS（临床时间序列），它将不规则的EHR轨迹转换为人类可读、紧凑、源可溯的树路径证据单元，无需患者级别的摘要或推理时的自回归解码。TreeText-CTS通过冻结的XGBoost模型路由多尺度窗口摘要，并将激活的树路径表述为由阈值条件组成的确定性、源可溯的证据单元。一个证据选择器从这些单元中选取信息量最大的子集，然后由语言模型编码器整合用于预测。在PhysioNet 2012死亡率、MIMIC-III死亡率和PhysioNet 2019脓毒症发作预测任务中，TreeText-CTS在评估的基于文本的EHR时间序列界面中取得了最佳的AUROC和AUPRC，相比先前最强的基于文本的界面，AUPRC绝对百分点提升了6.0至9.7，同时与数值时间序列模型保持竞争力。消融实验表明，树路径证据构建、证据选择和语言模型组成均对性能有贡献。由于传递给语言模型编码器的每个片段都由激活的树路径阈值条件构建而成，TreeText-CTS使得提供给最终预测器的证据可检查和源可溯。

## Abstract
Numerical time-series models can effectively process irregular electronic health record (EHR) trajectories, but they do not naturally expose the measurements and temporal patterns supporting each risk estimate as readable evidence. Existing text-based interfaces improve readability, but typically rely on either raw serialization, which is lengthy and redundant, or patient-level free-form summaries, which are difficult to trace to source measurements and time windows. To bridge this gap, we introduce TreeText-CTS (Clinical Time-Series), which converts irregular EHR trajectories into human-readable, compact, source-traceable tree-path evidence units without patient-level summarization or inference-time autoregressive decoding. TreeText-CTS routes multi-scale window summaries through frozen XGBoost models and verbalizes activated tree paths as deterministic, source-traceable evidence units composed of threshold conditions. An evidence selector assembles an informative subset of these units, which a language-model encoder then integrates for prediction. Across PhysioNet 2012 mortality, MIMIC-III mortality, and PhysioNet 2019 sepsis-onset forecasting, TreeText-CTS achieves the best AUROC and AUPRC among evaluated text-based EHR time-series interfaces, improving AUPRC by 6.0 to 9.7 absolute percentage points over the strongest prior text-based interface while remaining competitive with numerical time-series models. Ablations show that tree-path evidence construction, evidence selection, and language-model composition each contribute to performance. Because every span passed to the language-model encoder is constructed from activated tree-path threshold conditions, TreeText-CTS makes the evidence supplied to the final predictor inspectable and source-traceable.

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

#### 2.1 核心思想

- 利用 XGBoost 树模型从多尺度窗口摘要中学习预测相关的**阈值条件**（如“最小MAP ≤ 65”），将每个激活的根到叶子路径映射为**确定性、源可溯的文本证据单元**，然后通过一个学习到的紧凑证据选择器（Compact Evidence Selector, CES）从大量候选单元中选择最有信息量的子集，最后用临床语言模型编码器（BioClinical ModernBERT）整合这些单元进行预测。

#### 2.2 关键技术细节

1. **多尺度窗口摘要**：对每个病人和预测时间点，使用 **7 个回溯窗口**（1h、2h、4h、8h、16h、32h、48h）分别计算每个变量的 **9 个固定统计量**：最后值、均值、标准差、最小值、最大值、计数、净变化、距上次观测时间、缺失标志。

2. **树路径证据构建**：
   - 对每个窗口大小 \(W\)，单独训练一个 **XGBoost 模型**（在训练集上，输入为窗口摘要→输出二分类概率）。
   - 树模型 **冻结** 后，每个病人激活的每条根到叶子路径被转换为文本：例如 `"MAP_min <= 65; HR_last > 110"` → `"minimum MAP is at most 65; last HR is higher than 110"`。
   - 每条证据还附带 **源元组** \((t, W, tree\_id, leaf\_id)\)，保证完全可追溯。
   - 可选地，离线用 LLM（Qwen3.5-27B）为叶子添加**临床注释**（如“与低血压、心动过速和乳酸升高一致”），但不替代确定性谓词文本。

3. **紧凑证据选择器（CES）**：
   - 每个候选证据单元 \(e_{ij}\) 用一个 **72维向量** 表示：64维文本嵌入（Qwen3-Embedding-8B 投射）+ 8维标量元数据（时间新鲜度、叶子分数、支持数、窗口大小、文本长度、是否有临床注释等）。
   - 轻量 Transformer 对候选单元编码，线性门控网络输出**选择概率** \(q_{ij}\)（通过 sigmoid）。
   - **自批判策略梯度**：采样动作集 \(S^s\) 和贪心基线集 \(S^g\)，以任务损失差异作为优势，训练选择器。同时设置最小选择数 \(K_{min}=5\) 和最终预算 \(K=30\)。

4. **LM 分类器**：将静态协变量前缀（如年龄、性别）与排序后的选定证据单元拼接，送入 **BioClinical ModernBERT** 编码，加上线性分类头进行二分类预测（公式1）。

#### 2.3 公式或算法流程（文字说明）

- 训练分为两步：先冻结 XGBoost 树模型并缓存叶子证据；然后联合训练 **投影层、CES 和 LM 分类器**，优化目标为任务损失（BCE）+ 选择器策略梯度损失 + 熵正则化项。
- 推理时：给定病人轨迹 → 多尺度窗口摘要 → 路由 XGBoost 获得候选叶子 → TEM 映射为证据文本 → CES 选择至多 \(K\) 个 → 拼接前缀 → LM 分类器输出概率。

### 8. 不足与局限

- **实验覆盖范围**：仅在三个公开ICU数据集上评估，主要集中在院内死亡率和脓毒症预测，未涉及其他临床任务（如住院时间预测、再入院预测）或非ICU场景（如门诊、急诊）。方法在更广泛临床任务上的泛化性尚未验证。
- **依赖预先训练的树模型**：树路径证据完全由固定窗口的XGBoost模型决定，任何未在训练集中出现的模式（如罕见变量组合或跨窗口交互）可能永远无法被选择，限制了模型发现新 signal 的能力。
- **临床注释的质量风险**：离线 LLM 生成的临床注释（gloss）虽然不修改确定性谓词，但可能包含不准确或误导性信息，且仅部分叶子被标记（g=1 比例约32%~54%），注释质量未做严格一致性评估。
- **选择器训练复杂性**：使用自批判策略梯度训练离散选择，需要额外超参数（如 \(\lambda_{sel}\)、熵系数），训练可能更敏感；最小选择底限（\(K_{min}\)）的设定防止了坍塌，但也引入额外调节点。
- **可审计性 ≠ 因果解释**：构造级可审计性保证每个证据单元可追溯到源，但不保证证明这些条件构成因果解释，也不保证公平性或校准性，不适用于直接临床决策。
- **资源消耗**：虽然推理时较快，但训练仍需较多 GPU 时间（尤其是离线叶子嵌入计算和选择器训练），可能对资源有限的机构构成门槛。
- **未与其他可解释模型直接比较**：文中未与基于注意力的数值模型（如沙普利值或显著性图）进行可解释性方面的对比，仅与文本接口比较。
