ABSTRACT 25 

Summary:  The druggable genome encompasses the genes that are known or predicted to 26 

interact with drugs.  The Drug -Gene Interaction Database (DGIdb) provides an integrated resource 27 

for discovering and contextualizing these interactions, supporting a broad range of research and 28 

clinical applications . DGIdb is currently accessed through structured web interfaces and API calls, 29 

requiring users to translate natural -language questions into database -specific query patterns.  To 30 

allow for the use of DGIdb  through natural language, we developed the DGIdb Model Context 31 

Protocol (MCP) server, which allows large language models (LLMs) access to up -to -date 32 

information through the DGIdb API.  We demonstrate that the MCP server greatly enhances an 33 

LLM’s ability to answer questions requiring accurate, up -to -date biomedical knowledge drawn 34 

from structured external resources. 35 

Availability and implementation: The DGIdb MCP server is detailed at 36 

https://github.com/griffithlab/dgidb -mcp -server  and includes instructions for accessing the server 37 

through the Claude desktop app. 38

1. Introduction 39 

Drug -gene interactions play a central role in precision medicine, as they help determine 40 

which treatments are likely to be effective for a given patient and which may lead to resistance or 41 

adverse effects. As large -scale molecular studies increasingly reveal large sets of genes 42 

associated with disease, researchers need tools that can connect these findings to potential 43 

therapeutic strategies. The Drug -Gene Interaction Database (DGIdb) 1–5 addresses this need by 44 

aggregating and harmonizing drug -gene interaction information from multiple sources, providing 45 

a centralized resource for assessing gene druggability and therapeutic relevance. DGIdb supports 46 

downstream research and clinical -informatics applications, including gene -drug linking for 47 

prioritized disease genes 6, exploratory hypothesis generation 7, drug repurposing 8, genomic gene -48 

list annotation 9, and precision oncology workflows 10 . The value of searching a catalog of drug -49 

gene interactions is well illustrated by the challenge of FLT3 -mutated Acute Myeloid Leukemia .50 

Although the FLT3 inhibitor Gilteritinib is initially effective, patients often develop resistance due 51 

to additional genetic changes that emerge during treatment. 11  In such settings, DGIdb could be 52 

used to identify drugs known or predicted to interact with genes implicated in resistance, 53 

supporting exploratory analyses and prioritization of alternative therapeutic candidates. 54 

Despite DGIdb’s utility, this process can be time -consuming in multistep analyses, such 55 

as studies of acquired drug resistance, where users must first identify resistance -associated 56 

genes, then perform separate queries for each gene, and finally manually combine and prioritize 57 

potentially large lists of interacting drugs. As a result, DGIdb is difficult to use directly within 58 

natural -language workflows and is largely inaccessible to large language models (LLMs), which 59 

cannot natively construct or execute these structured queries. At the same time, LLMs have 60 

emerged as promising tools for a wide range of biomedical applications, including literature 61 

summarization and clinical decision support, due to their ability to synthesize complex information 62 

and respond flexibly to natural -language questions. 12 –16  However, LLMs rely on static internal 63 

knowledge, may generate inaccurate or unsupported statements, and cannot directly access 64

To address these challenges, the Model Context Protocol (MCP) provides a standardized 67 

way to connect LLMs to authoritative external data sources. MCP servers serve as a bridge 68 

between the flexibility of LLMs and the precision of curated biomedical databases and have been 69 

increasingly applied to biomedical domain APIs, databases, and literature search endpoints, 70 

including variant -level knowledgebases via the CIViC MCP server. 18 –20  The DGIdb MCP server 71 

builds upon this framework, allowing LLMs and researchers to retrieve, interpret, and cite drug -72 

gene interaction information directly from the curated resources, making it easier to integrate 73 

accurate interaction data into genome -wide analyses and therapeutic decision -making. 74 

2. DGIdb MCP Server Description 75 

The DGIdb MCP server is publicly available through Cloudflare. The server provides four 76 

tools, each corresponding to predefined GraphQL queries for integration with LLMs. These tools 77 

allow LLMs to retrieve drug information, gene information, and drug -gene interactions based on 78 

either a supplied gene list or drug list. The drug information tool returns key attributes such as 79 

FDA approval status, drug classifications including immunotherapy and antineoplastic drugs, and 80 

additional characteristics such as drug class. The gene information tool provides DGIdb gene 81 

category annotations, such as kinase, ligase, and nuclear receptor designations, along with other 82 

attributes relevant to gene druggability. Together, these data sources supply structured and well -83 

curated information that supplements an LLM’s understanding of the drug -gene interaction space. 84 

When the LLM calls the corresponding interaction query with a supplied list of genes or drugs, 85 

the server returns a ranked list of their known interactions. Each interaction includes an interaction 86 

type and, when applicable, directionality (activating or inhibitory) which together describe the 87 

mechanism of action and effect on biological activity. Every interaction is also linked to its 88 

supporting evidence, which may come from a publication or a curated database. Interactions are 89 

ranked primarily according to the FDA approval status of the associated drugs, and secondarily 90

gene compared to global averages. The final score prioritizes interactions that are both well 94 

supported and more specific, and remains static within a data release. This ranking strategy 95 

allows LLMs to prioritize clinically informative and well -supported interactions. 96 

To accommodate spelling variations and alternate names, the MCP server normalizes 97 

drug and gene inputs to the closest matching label recognized by DGIdb. Alias similarity is scored 98 

using Dice’s coefficient over bigrams, which requires limited memory making it ideal for Cloudflare 99 

workers. The MCP server obtains drug and gene aliases from the VICC Normalization Services, 100 

which provide a standardized set of identifiers and synonyms for use during the scoring 101 

process. 21,22

3.  Example Use Case s103 

1. A user might ask, “Which FDA -approved drugs target the gene KIT , and what is the 104 

supporting evidence?” In response, the LLM identifies the gene of interest and the user’s intent, 105 

then invokes the appropriate MCP tool. The MCP server normalizes the gene name and issues 106 

the corresponding GraphQL query to the DGIdb API. It returns a ranked list of drugs that interact 107 

with KIT , along with interaction metadata such as interaction score, interaction type and direction, 108 

FDA -approval status, and supporting sources. Using this structured output, the LLM composes a 109

4. Benchmarking LLM Performance with DGIdb and CIViC MCP Servers 126 

We developed a set of classification and information -retrieval tasks to measure how 127 

effectively the MCP server enhances an LLM’s ability to answer questions requiring accurate and 128 

current biomedical knowledge. As a first task, we asked GPT -5 to label drugs according to three 129 

DGIdb -derived attributes: FDA approval status, immunotherapy classification, and antineoplastic 130 

classification (Supplementary Table 3). We selected 100 drug names using stratified sampling to 131 

approximate a 50/50 balance of positive and negative examples for each label. Because relatively 132 

few drugs in DGIdb are classified as immunotherapies, achieving a perfectly even split was not 133 

possible. GPT -5 was prompted to return three comma -separated YES/NO values to facilitate 134 

parsing (Supplementary Prompt 1). Across all labels, GPT -5 augmented with the DGIdb MCP 135 

substantially outperformed GPT -5 alone achieving a macro weighted F1 -score of 0.99 compared 136 

to 0.75. GPT -5 without MCP struggled particularly with identifying immunotherapies, producing 137 

many false negatives and yielding a recall of only 0.38. 138 

Next, we compared how often GPT -5 would invoke the DGIdb MCP server when 139 

requesting the same drug information without explicitly mentioning DGIdb (Supplementary 140 

Prompt 2). The DGIdb MCP server was available to the model in the same configuration as the 141 

previous task, but tool use was not explicitly instructed. GPT -5 did not use the DGIdb MCP server 142

5. Conclusion 175 

We demonstrate that the DGIdb MCP server enables reliable, structured exploration of 176 

drug -gene interactions by allowing an LLM to query a comprehensive, curated interaction 177 

resource in response to natural -language questions. By grounding GPT -5 in DGIdb, this approach 178 

supports accurate retrieval, ranking, and contextualization of interaction evidence, which is 179 

particularly important for tasks such as identifying alternative therapeutic candidates and 180 

interpreting resistance -associated genes. 181 

We further show that LLMs can make effective use of multiple MCP servers 182 

simultaneously. In a drug -candidate identification task for therapy resistance, GPT -5 successfully 183 

combined variant -level evidence retrieved from the CIViC MCP server with drug -gene interaction 184 

data from the DGIdb MCP server, demonstrating how MCP servers can be composed to support 185 

multi -step biomedical reasoning workflows that require information from multiple sources. 186 

Our evaluation also highlights an important practical consideration: GPT -5 did not 187 

consistently invoke MCP servers unless the relevant biomedical resource is mentioned explicitly 188 

in the prompt. When the source (e.g., “DGIdb”) is named, GPT -5 reliably delegates retrieval to 189 

the MCP server. When it is omitted, however, the model often defaults to its internal knowledge, 190 

resulting in reduced accuracy, particularly for identifying immunotherapies. This finding 191 

emphasizes the importance of prompt design in ensuring consistent MCP usage. 192 

While the MCP framework has previously been applied to variant -level knowledge through 193 

the CIViC MCP server 20 , this work extends the paradigm to the drug -gene interaction domain and 194
