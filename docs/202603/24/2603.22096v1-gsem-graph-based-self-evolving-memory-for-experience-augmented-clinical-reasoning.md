## 摘要
临床决策智能体可以从重用先前的决策经验中获益。然而，许多记忆增强方法将经验存储为独立的记录，缺乏显式的关系结构，这可能导致检索噪声、重用不可靠，在某些情况下甚至比直接的大语言模型（LLM）推理性能更差。我们提出了 GSEM（基于图的自演化记忆），这是一个临床记忆框架，它将临床经验组织成双层记忆图，既捕捉了每个经验内部的决策结构，也捕捉了经验之间的关系依赖，并支持适用性感知检索以及基于在线反馈的节点质量和边权重校准。在 MedR-Bench 和 MedAgentsBench 两个基准测试中，结合两种 LLM 主干网络，GSEM 在所有基线方法中均取得了最高的平均准确率，在使用 DeepSeek-V3.2 和 Qwen3.5-35B 时分别达到了 70.90% 和 69.24%。代码可在 https://github.com/xhan1022/gsem 获取。

## Abstract
Clinical decision-making agents can benefit from reusing prior decision experience. However, many memory-augmented methods store experiences as independent records without explicit relational structure, which may introduce noisy retrieval, unreliable reuse, and in some cases even hurt performance compared to direct LLM inference. We propose GSEM (Graph-based Self-Evolving Memory), a clinical memory framework that organizes clinical experiences into a dual-layer memory graph, capturing both the decision structure within each experience and the relational dependencies across experiences, and supporting applicability-aware retrieval and online feedback-driven calibration of node quality and edge weights. Across MedR-Bench and MedAgentsBench with two LLM backbones, GSEM achieves the highest average accuracy among all baselines, reaching 70.90\% and 69.24\% with DeepSeek-V3.2 and Qwen3.5-35B, respectively. Code is available at https://github.com/xhan1022/gsem.

*   **研究动机**：临床决策具有高风险、信息不完整和强条件依赖的特点。虽然大语言模型（LLM）具备医学知识，但在处理复杂病例时，仅靠参数化知识不足以应对多变的临床模式。
*   **现有挑战**：目前的记忆增强方法（如RAG）通常将经验存储为孤立的记录，缺乏显式的关系结构。这导致了两种典型的失败模式：
    1.  **边界失效（Boundary Failure）**：检索到的经验仅部分匹配，忽略了关键的边界条件。
    2.  **协作失效（Collaboration Failure）**：检索到多个相互冲突或缺乏共同使用价值的经验，导致推理逻辑混乱。
*   **核心目标**：提出一种能够捕捉经验内部决策结构及经验间关联，并能根据反馈动态进化的临床记忆框架。

### 2. 方法论
GSEM 框架由三个阶段组成：记忆构建、记忆检索和记忆演化。
*   **经验定义**：将经验形式化为四元组 $(c_i, s_i, z_i, Q_i)$，包括适用条件、决策策略、极性标签（**适应症 Indication** 或 **禁忌症 Contraindication**）以及质量评分。
*   **双层记忆图结构**：
    *   **实体层 (Entity Layer)**：将每个经验分解为具有语义角色（条件、约束、行动、理由、结果）的节点，捕捉经验内部的决策流。
    *   **经验层 (Experience Layer)**：节点为完整经验，边权重 $W$ 表示经验间的关联强度（基于实体重叠、结构相似性、协同价值和任务一致性计算）。
*   **适用性感知检索**：
    *   **混合种子召回**：结合基于实体的稀疏检索和基于嵌入的稠密检索，生成初始种子集。
    *   **多种子图遍历**：由 LLM 引导在经验层图上进行迭代遍历（动作包括收集、探索、回溯、停止），以寻找满足边界条件且具有协同价值的经验组合。
*   **自演化机制**：
    *   **在线校准**：任务完成后，根据结果反馈（标量信号 $\Delta_t$）动态更新节点质量 $Q$ 和边权重 $W$。
    *   **信用分配**：通过指数衰减的排名权重将反馈分配给被激活的节点和边，使高价值经验被强化，失败相关的关联被削弱。

*   **自演化的有效性**：在线反馈机制能显著极化经验质量分布，使高价值经验更易被检索，从而在不修改模型参数的情况下持续优化性能。
*   **模型不对称性**：性能主要受生成器（Generator）能力影响，而检索器（Retriever）可以使用较小的模型（如 Qwen-35B）而不显著损失精度，这为低成本部署提供了可能。

### 7. 优点/亮点
*   **结构化建模**：通过双层图结构解决了传统 RAG 难以处理的“经验间冲突”和“边界模糊”问题。
*   **禁忌症学习**：显式提取并利用失败经验（Contraindications），这在安全性至关重要的医学领域具有极高的实用价值。
*   **动态校准**：无需微调模型，通过权重更新实现记忆的“新陈代谢”，保证了系统的长期可靠性。

### 8. 不足与局限
*   **计算开销**：记忆构建阶段需要采样多条推理轨迹并进行可靠性验证，图遍历检索也比简单向量检索更耗时。
*   **提示词敏感性**：LLM 引导的图遍历策略可能对 Prompt 较为敏感，且在路径选择上存在一定的随机性。
*   **现实差距**：虽然在公共基准上表现优异，但尚未在真实的临床工作流（如交互式决策、长期随访）中进行验证。
